GradleгӮ’дҪҝз”ЁгҒ—гҒҰгҖҒж–°гҒ—гҒ„гғ—гғӯгӮёгӮ§гӮҜгғҲгӮ’дҪңжҲҗгҒ—гҖҒгӮЁгӮҜгӮ№гғҶгғігӮ·гғ§гғігӮ’иҝҪеҠ гҒҫгҒҹгҒҜеүҠйҷӨгҒ—гҖҒй–ӢзҷәгғўгғјгғүгӮ’иө·еӢ•гҒ—гҖҒгӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігӮ’гғҮгғҗгғғгӮ°гҒ—гҖҒгӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігӮ’jarгҖҒгғҚгӮӨгғҶгӮЈгғ–е®ҹиЎҢгғ•гӮЎгӮӨгғ«гҖҒгҒҫгҒҹгҒҜгӮігғігғҶгғҠгғ•гғ¬гғігғүгғӘгғјгҒӘе®ҹиЎҢгғ•гӮЎгӮӨгғ«гҒ«гғ“гғ«гғүгҒ—гҒҫгҒҷгҖӮGradleгғ—гғӯгӮёгӮ§гӮҜгғҲгҒ®гғЎгӮҝгғҮгғјгӮҝгӮ’дҪҝз”ЁгҒ—гҒҰгҖҒгҒҠж°—гҒ«е…ҘгӮҠгҒ®IDEгҒ«гғ—гғӯгӮёгӮ§гӮҜгғҲгӮ’гӮӨгғігғқгғјгғҲгҒ—гҒҫгҒҷгҖӮ

ж–°иҰҸгғ—гғӯгӮёгӮ§гӮҜгғҲгҒ®дҪңжҲҗ

Gradleгғ—гғӯгӮёгӮ§гӮҜгғҲгӮ’з”ҹжҲҗгҒҷгӮӢгҒ«гҒҜгҖҒ Quarkus CLIгҒҫгҒҹгҒҜQuarkus Mavenгғ—гғ©гӮ°гӮӨгғігӮ’дҪҝз”ЁгҒ§гҒҚгҒҫгҒҷ:

CLI
quarkus create app my-groupId:my-artifactId \
    --extensions=rest,rest-jackson \
    --gradle

Quarkus CLIгҒ®гӮӨгғігӮ№гғҲгғјгғ«ж–№жі•гӮ„дҪҝз”Ёж–№жі•гҒ«гҒӨгҒ„гҒҰгҒҜгҖҒ Quarkus CLIгӮ¬гӮӨгғү гӮ’еҸӮз…§гҒ—гҒҰгҒҸгҒ гҒ•гҒ„гҖӮ

Maven
mvn io.quarkus.platform:quarkus-maven-plugin:3.15.1:create \
    -DprojectGroupId=my-groupId \
    -DprojectArtifactId=my-artifactId \
    -Dextensions="rest,rest-jackson" \
    -DbuildTool=gradle
mvn io.quarkus:quarkus-maven-plugin:3.15.1:create гӮ’иө·еӢ•гҒҷгӮӢгҒЁгҖҒMaven гғ—гғ©гӮ°гӮӨгғігҒҜгғҰгғјгӮ¶гғје…ҘеҠӣгӮ’иҰҒжұӮгҒ—гҒҫгҒҷгҖӮ -B гӮ’ Maven гӮігғһгғігғүгҒ«жёЎгҒҷгҒ“гҒЁгҒ§гҖҒгҒ“гҒ®еҜҫи©ұеһӢгғўгғјгғүгӮ’з„ЎеҠ№гҒ«гҒҷгӮӢгҒ“гҒЁгҒҢеҮәжқҘгҒҫгҒҷ (гғҮгғ•гӮ©гғ«гғҲеҖӨгӮ’дҪҝз”ЁгҒ—гҒҫгҒҷ)гҖӮ
Kotlin DSLгӮ’дҪҝз”ЁгҒ—гҒҹгҒ„е ҙеҗҲгҒҜгҖҒ gradle гҒ®д»ЈгӮҸгӮҠгҒ« gradle-kotlin-dsl гӮ’дҪҝз”ЁгҒ—гҒҰгҒҸгҒ гҒ•гҒ„гҖӮ

Quarkusгғ—гғӯгӮёгӮ§гӮҜгғҲгҒ®гӮ№гӮӯгғЈгғ•гӮ©гғјгғ«гғүгҒҜгҖҒгғ—гғӯгӮёгӮ§гӮҜгғҲгҒ«Gradleгғ©гғғгғ‘гғј( ./gradlew )гӮ’иҮӘеӢ•зҡ„гҒ«гӮӨгғігӮ№гғҲгғјгғ«гҒ—гҒҫгҒҷгҖӮ

гӮ№гӮҝгғігғүгӮўгғӯгғігҒ®GradleгӮ’дҪҝз”ЁгҒ—гҒҹгҒ„е ҙеҗҲгҒҜгҖҒGradle 8.9 гӮ’дҪҝз”ЁгҒ—гҒҰгҒҸгҒ гҒ•гҒ„гҖӮ

гғ—гғӯгӮёгӮ§гӮҜгғҲгҒҜгҖҒжёЎгҒ•гӮҢгҒҹartifactIdгҒ«гҒЎгҒӘгӮ“гҒ еҗҚеүҚгҒ®гғҮгӮЈгғ¬гӮҜгғҲгғӘгҒ«з”ҹжҲҗгҒ•гӮҢгҒҫгҒҷгҖӮ

гғҚгӮӨгғҶгӮЈгғ–гғўгғјгғүгҒЁJVMгғўгғјгғүз”ЁгҒ®дёҖеҜҫгҒ®DockerfilesгӮӮ src/main/docker гҒ§з”ҹжҲҗгҒ•гӮҢгҒҫгҒҷгҖӮгҒ“гӮҢгӮүгҒ®DockerfileгҒ«гҒҜгҖҒгӮӨгғЎгғјгӮёгӮ’гғ“гғ«гғүгҒ—гҒҰгӮігғігғҶгғҠгӮ’е®ҹиЎҢгҒҷгӮӢжүӢй ҶгҒҢжӣёгҒӢгӮҢгҒҰгҒ„гҒҫгҒҷгҖӮ

JVMгғўгғјгғүгҒ§гҒ®гӮ«гӮ№гӮҝгғ гғҶгӮ№гғҲж§ӢжҲҗгғ—гғӯгғ•гӮЎгӮӨгғ«

гғҮгғ•гӮ©гғ«гғҲгҒ§гҒҜгҖҒJVMгғўгғјгғүгҒ®QuarkusгғҶгӮ№гғҲгҒҜгҖҒ test иЁӯе®ҡгғ—гғӯгғ•гӮЎгӮӨгғ«гӮ’дҪҝз”ЁгҒ—гҒҰе®ҹиЎҢгҒ•гӮҢгҒҫгҒҷгҖӮQuarkusгҒ®иЁӯе®ҡгғ—гғӯгғ•гӮЎгӮӨгғ«гҒ«ж…ЈгӮҢгҒҰгҒ„гҒӘгҒ„е ҙеҗҲгҒҜгҖҒ иЁӯе®ҡгғ—гғӯгғ•гӮЎгӮӨгғ« гҒ®гғүгӮӯгғҘгғЎгғігғҲгҒ§еҝ…иҰҒгҒӘжғ…е ұгҒҢгҒҷгҒ№гҒҰиӘ¬жҳҺгҒ•гӮҢгҒҰгҒ„гҒҫгҒҷгҒ®гҒ§гҖҒгҒқгҒЎгӮүгӮ’еҸӮз…§гҒ—гҒҰгҒҸгҒ гҒ•гҒ„гҖӮ

гҒ—гҒӢгҒ—гҖҒд»ҘдёӢгҒ«зӨәгҒҷ Gradle гғ“гғ«гғүиЁӯе®ҡгҒ§гғҶгӮ№гғҲз”ЁгҒ®гӮ«гӮ№гӮҝгғ иЁӯе®ҡгғ—гғӯгғ•гӮЎгӮӨгғ«гӮ’дҪҝз”ЁгҒҷгӮӢгҒ“гҒЁгҒҜеҸҜиғҪгҒ§гҒҷгҖӮгҒ“гӮҢгҒҜдҫӢгҒҲгҒ°гҖҒгғҮгғ•гӮ©гғ«гғҲгҒ®гғҶгӮ№гғҲгғҮгғјгӮҝгғҷгғјгӮ№гҒ§гҒҜгҒӘгҒ„зү№е®ҡгҒ®гғҮгғјгӮҝгғҷгғјгӮ№гӮ’дҪҝгҒЈгҒҰгғҶгӮ№гғҲгӮ’е®ҹиЎҢгҒҷгӮӢеҝ…иҰҒгҒҢгҒӮгӮӢе ҙеҗҲгҒ«дҫҝеҲ©гҒ§гҒҷгҖӮ

    systemProperty "quarkus.test.profile", "foo" (1)

д»ҠгҒ®гҒЁгҒ“гӮҚгҖҒгғҚгӮӨгғҶгӮЈгғ–гғўгғјгғүгҒ§гӮ«гӮ№гӮҝгғ гғҶгӮ№гғҲиЁӯе®ҡгғ—гғӯгғ•гӮЎгӮӨгғ«гӮ’дҪҝз”ЁгҒҷгӮӢгҒ“гҒЁгҒҜгҒ§гҒҚгҒҫгҒӣгӮ“гҖӮгғҚгӮӨгғҶгӮЈгғ–гғҶгӮ№гғҲгҒҜеёёгҒ« prod гғ—гғӯгғ•гӮЎгӮӨгғ«гӮ’дҪҝз”ЁгҒ—гҒҰе®ҹиЎҢгҒ•гӮҢгҒҫгҒҷгҖӮ

гҒ“гӮҢгҒ«гӮҲгӮҠгҖҒе®ҹйҡӣгҒ«гӮўгғ—гғӘгӮ’е®ҹиЎҢгҒҷгӮӢгҒ®гҒЁеҗҢгҒҳз’°еўғгҒ§гҖҒеҗҢгҒҳгӮөгғјгғ“гӮ№гҒ«гӮўгӮҜгӮ»гӮ№гҒ—гҒӘгҒҢгӮүй–ӢзҷәгҒҷгӮӢгҒ“гҒЁгҒҢгҒ§гҒҚгҒҫгҒҷгҖӮ

жң¬з•Әз’°еўғгҒ§гҒҜдҪҝз”ЁгҒ—гҒӘгҒ„гҒ§гҒҸгҒ гҒ•гҒ„гҖӮй–Ӣзҷәз’°еўғгҒ§гҒ®гҒҝдҪҝз”ЁгҒ—гҒҰгҒҸгҒ гҒ•гҒ„гҖӮжң¬з•Әз’°еўғгҒ®гӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігӮ’й–ӢзҷәгғўгғјгғүгҒ§е®ҹиЎҢгҒ—гҒҰгҒҜгҒ„гҒ‘гҒҫгҒӣгӮ“гҖӮ

гҒқгҒ®гҒҹгӮҒгҒ«гҒҜгҖҒ mutable-jar гҒ®гғ•гӮ©гғјгғһгғғгғҲгӮ’дҪҝгҒЈгҒҰ mutable гӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігӮ’ж§ӢзҜүгҒҷгӮӢеҝ…иҰҒгҒҢгҒӮгӮҠгҒҫгҒҷгҖӮ application.properties гҒ§д»ҘдёӢгҒ®гғ—гғӯгғ‘гғҶгӮЈгӮ’иЁӯе®ҡгҒ—гҒҫгҒҷгҖӮ

1 гҒ“гӮҢгҒҜгҖҒQuarkusгҒ«Mutable-jarеҪўејҸгӮ’дҪҝз”ЁгҒҷгӮӢгӮҲгҒҶгҒ«жҢҮзӨәгҒ—гҒҫгҒҷгҖӮгғҹгғҘгғјгӮҝгғ–гғ«гӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігҒ«гҒҜгҖҒQuarkusгҒ®гғҮгғ—гғӯгӮӨгғЎгғігғҲжҷӮй–“гҒ®йғЁеҲҶгӮӮеҗ«гҒҫгӮҢгҒҰгҒ„гӮӢгҒҹгӮҒгҖҒгӮҲгӮҠеӨҡгҒҸгҒ®гғҮгӮЈгӮ№гӮҜгӮ№гғҡгғјгӮ№гӮ’еҚ жңүгҒ—гҒҫгҒҷгҖӮжҷ®йҖҡгҒ«е®ҹиЎҢгҒ—гҒҹе ҙеҗҲгҒҜгҖҒгӮӨгғҹгғҘгғјгӮҝгғ–гғ«гӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігҒЁеҗҢгҒҳйҖҹеәҰгҒ§иө·еӢ•гҒ—гҖҒеҗҢгҒҳгғЎгғўгғӘгғјгӮ’дҪҝз”ЁгҒ—гҒҫгҒҷгҒҢгҖҒdevгғўгғјгғүгҒ§иө·еӢ•гҒҷгӮӢгҒ“гҒЁгӮӮгҒ§гҒҚгҒҫгҒҷгҖӮ
2 гғӘгғўгғјгғҲеҒҙгҒЁгғӯгғјгӮ«гғ«еҒҙгҒ®йҖҡдҝЎгӮ’е®үе…ЁгҒ«дҝқгҒӨгҒҹгӮҒгҒ®гғ‘гӮ№гғҜгғјгғүгҒ§гҒҷгҖӮ
3 гӮўгғ—гғӘгҒҢdevгғўгғјгғүгҒ§е®ҹиЎҢгҒ•гӮҢгӮӢURLгҖӮгҒ“гӮҢгҒҜгғӯгғјгӮ«гғ«еҒҙгҒ§гҒ®гҒҝеҝ…иҰҒгҒӘгҒ®гҒ§гҖҒгғ—гғӯгғ‘гғҶгӮЈгғј гғ•гӮЎгӮӨгғ«гҒӢгӮүйҷӨеӨ–гҒ—гҒҰгҖҒгӮігғһгғігғү гғ©гӮӨгғігҒ§гӮ·гӮ№гғҶгғ  гғ—гғӯгғ‘гғҶгӮЈгғјгҒЁгҒ—гҒҰжҢҮе®ҡгҒҷгӮӢгҒЁгӮҲгҒ„гҒ§гҒ—гӮҮгҒҶгҖӮ

If you plan on running the application via Docker, then you’ll need to add -e QUARKUS_LAUNCH_DEVMODE=true to the docker run command. When the application starts you should now see the following line in the logs: Profile dev activated. Live Coding activated. You will also need to give the application the rights to update the deployment resources by adding RUN chmod o+rw -R /deployments after the COPY commands into your Dockerfile. For security reasons, this option should not be added to the production Dockerfile.

гғҚгӮӨгғҶгӮЈгғ–е®ҹиЎҢеҸҜиғҪгғ•гӮЎгӮӨгғ«гҒҜгҖҒгҒҠдҪҝгҒ„гҒ®гӮӘгғҡгғ¬гғјгғҶгӮЈгғігӮ°гӮ·гӮ№гғҶгғ гҒ«еӣәжңүгҒ®гӮӮгҒ®гҒ«гҒӘгӮҠгҒҫгҒҷгҖӮгӮігғігғҶгғҠгғјеҶ…гҒ§е®ҹиЎҢгҒ•гӮҢгӮӢе®ҹиЎҢгғ•гӮЎгӮӨгғ«гӮ’дҪңжҲҗгҒҷгӮӢгҒ«гҒҜгҖҒж¬ЎгҒ®гӮҲгҒҶгҒ«гҒ—гҒҫгҒҷгҖӮ

гғҮгғ•гӮ©гғ«гғҲгҒ§гҒҜгҖҒгғҚгӮӨгғҶгӮЈгғ–е®ҹиЎҢеҸҜиғҪгғ•гӮЎгӮӨгғ«гҒҜ quay.io/quarkus/ubi-quarkus-mandrel-builder-image:jdk-21 DockerгӮӨгғЎгғјгӮёгӮ’дҪҝз”ЁгҒ—гҒҰз”ҹжҲҗгҒ•гӮҢгҒҫгҒҷгҖӮ

з•°гҒӘгӮӢDockerгӮӨгғЎгғјгӮёгҒ§гғҚгӮӨгғҶгӮЈгғ–е®ҹиЎҢеҸҜиғҪгғ•гӮЎгӮӨгғ«гӮ’гғ“гғ«гғүгҒ—гҒҹгҒ„е ҙеҗҲ(дҫӢгҒҲгҒ°гҖҒз•°гҒӘгӮӢGraalVMгҒ®гғҗгғјгӮёгғ§гғігӮ’дҪҝз”ЁгҒҷгӮӢгҒӘгҒ©)гҖҒ -Dquarkus.native.builder-image=<image name> гҒ®buildеј•ж•°гӮ’дҪҝз”ЁгҒ—гҒҰгҒҸгҒ гҒ•гҒ„гҖӮ

гғҚгӮӨгғҶгӮЈгғ–гғҶгӮ№гғҲгҒ®е®ҹиЎҢ

ж¬ЎгҒ®гӮҲгҒҶгҒ«гғҚгӮӨгғҶгӮЈгғ–гғҶгӮ№гғҲгӮ’е®ҹиЎҢгҒ—гҒҫгҒҷ:

гҒ“гҒ®гӮҝгӮ№гӮҜгҒҜ quarkusBuild гҒ«дҫқеӯҳгҒ—гҒҰгҒ„гӮӢгҒ®гҒ§гҖҒгғҶгӮ№гғҲгӮ’е®ҹиЎҢгҒҷгӮӢеүҚгҒ«гғҚгӮӨгғҶгӮЈгғ–гӮӨгғЎгғјгӮёгӮ’з”ҹжҲҗгҒ—гҒҫгҒҷгҖӮ

гғҮгғ•гӮ©гғ«гғҲгҒ§гҒҜгҖҒ native-test гҒ®гӮҪгғјгӮ№гӮ»гғғгғҲгҒҜгҖҒ main гҒЁ test гҒ®гӮҪгғјгӮ№гӮ»гғғгғҲгҒ«еҹәгҒҘгҒ„гҒҰгҒ„гҒҫгҒҷгҖӮгӮҪгғјгӮ№гӮ»гғғгғҲгӮ’иҝҪеҠ гҒҷгӮӢгҒ“гҒЁгӮӮеҸҜиғҪгҒ§гҒҷгҖӮгҒҹгҒЁгҒҲгҒ°гҖҒзөұеҗҲгғҶгӮ№гғҲгҒҢ integrationTest гҒ®гӮҪгғјгӮ№гӮ»гғғгғҲгҒ«гҒӮгӮӢе ҙеҗҲгҒҜгҖҒж¬ЎгҒ®гӮҲгҒҶгҒ«жҢҮе®ҡгҒ—гҒҫгҒҷгҖӮ

    }
}

зөҗеҗҲгғҶгӮ№гғҲгҒ®е®ҹиЎҢ

пјҲ @QuarkusIntegrationTest гҒ§гӮўгғҺгғҶгғјгӮ·гғ§гғігҒ•гӮҢгҒҹпјүQuarkusзөҗеҗҲгғҶгӮ№гғҲгҒҜгҖҒQuarkusгҒҢз”ҹжҲҗгҒ—гҒҹгӮўгғјгғҶгӮЈгғ•гӮЎгӮҜгғҲгҒ§е®ҹиЎҢгҒ•гӮҢгҒҫгҒҷгҖӮгҒ“гӮҢгӮүгҒ®гғҶгӮ№гғҲгҒҜгҖҒ src/integrationTest/java гғҮгӮЈгғ¬гӮҜгғҲгғӘгҒ«й…ҚзҪ®гҒ—гҖҒгҒқгӮҢгӮ’дҪҝз”ЁгҒ—гҒҰе®ҹиЎҢгҒҷгӮӢгҒ“гҒЁгҒҢгҒ§гҒҚгҒҫгҒҷгҖӮ

./gradlew quarkusIntTest

гҒ“гҒ®гӮҝгӮ№гӮҜгҒҜ test гҒЁ quarkusBuild гӮҝгӮ№гӮҜгҒ«дҫқеӯҳгҒ—гҒҰгҒ„гҒҫгҒҷгҖӮжңҖзөӮзҡ„гҒӘгӮўгғјгғҶгӮЈгғ•гӮЎгӮҜгғҲгҒҜгҖҒгғҶгӮ№гғҲгӮ’е®ҹиЎҢгҒҷгӮӢеүҚгҒ«з”ҹжҲҗгҒ•гӮҢгҒҫгҒҷгҖӮ

fast-jar гҒ®дҪҝз”Ё

fast-jar гҒҢгғҮгғ•гӮ©гғ«гғҲгҒ® quarkus гғ‘гғғгӮұгғјгӮёгӮҝгӮӨгғ—гҒ«гҒӘгӮҠгҒҫгҒ—гҒҹгҖӮ ./gradlew build гӮігғһгғігғүгҒ®зөҗжһңгҖҒ build гҒ®дёӢгҒ« quarkus-app гҒЁгҒ„гҒҶеҗҚеүҚгҒ®ж–°гҒ—гҒ„гғҮгӮЈгғ¬гӮҜгғҲгғӘгҒҢдҪңжҲҗгҒ•гӮҢгҒҫгҒҷгҖӮ

гӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігҒҜж¬ЎгҒ®гӮҲгҒҶгҒ«гҒ§е®ҹиЎҢгҒ§гҒҚгҒҫгҒҷ: java -jar target/quarkus-app/quarkus-run.jar

з”ҹжҲҗгҒ•гӮҢгҒҹ jar гӮ’жӯЈеёёгҒ«е®ҹиЎҢгҒҷгӮӢгҒҹгӮҒгҒ«гҒҜгҖҒ quarkus-app гғҮгӮЈгғ¬гӮҜгғҲгғӘгҒ®гҒҷгҒ№гҒҰгҒ®еҶ…е®№гҒҢеҝ…иҰҒгҒ§гҒҷгҖӮгҒ„гҒҡгӮҢгҒӢгҒ®гғ•гӮЎгӮӨгғ«гҒҢж¬ иҗҪгҒ—гҒҰгҒ„гӮӢгҒЁгҖҒгӮўгғ—гғӘгӮұгғјгӮ·гғ§гғігҒҢиө·еӢ•гҒ—гҒӘгҒӢгҒЈгҒҹгӮҠгҖҒжӯЈгҒ—гҒҸж©ҹиғҪгҒ—гҒӘгҒ„еҸҜиғҪжҖ§гҒҢгҒӮгӮҠгҒҫгҒҷгҖӮ
fast-jar гғ‘гғғгӮұгғјгӮёгғігӮ°гҒ§гҒҜгҖҒгҒ©гҒ®дҫқеӯҳй–ўдҝӮгҒ®jarгҒ«гӮҜгғ©гӮ№гӮ„гғӘгӮҪгғјгӮ№гҒҢеҗ«гҒҫгӮҢгҒҰгҒ„гӮӢгҒӢгҒЁгҒ„гҒҶжғ…е ұгҒҢгӮӨгғігғҮгғғгӮҜгӮ№еҢ–гҒ•гӮҢгҒҰгҒ„гӮӢгҒҹгӮҒгҖҒгғ¬гӮ¬гӮ·гғјгҒ®Quarkus jarгӮҲгӮҠгӮӮиө·еӢ•гҒҢе°‘гҒ—йҖҹгҒҸгҖҒгғЎгғўгғӘж¶ҲиІ»йҮҸгӮӮгӮҸгҒҡгҒӢгҒ«е°‘гҒӘгҒ„гӮўгғјгғҶгӮЈгғ•гӮЎгӮҜгғҲгӮ’дҪңжҲҗгҒ§гҒҚгҒҫгҒҷгҖӮгҒ“гҒ®гҒҹгӮҒгҖҒгӮҜгғ©гӮ№гӮ„гғӘгӮҪгғјгӮ№гӮ’гғӯгғјгғүгҒҷгӮӢйҡӣгҒ«гҖҒ гғ¬гӮ¬гӮ·гғј jar гҒҢеҝ…иҰҒгҒЁгҒҷгӮӢгӮҜгғ©гӮ№гғ‘гӮ№дёҠгҒ®гҒҷгҒ№гҒҰгҒ® jar гӮ’жӨңзҙўгҒҷгӮӢеҝ…иҰҒгҒҢгҒӘгҒҸгҒӘгӮҠгҒҫгҒҷгҖӮ

Gradle гӮӯгғЈгғғгӮ·гғігӮ° / гӮҝгӮ№гӮҜе…ҘеҠӣ

By default, system properties starting with quarkus. and environment variables, including those from ~/.env, starting with QUARKUS_, are considered as inputs for the Gradle tasks. This means that only changes to those system properties or environment variables will cause Gradle’s up-to-date to trigger a rebuild. Changes to other system properties or environment variables do not change Quarkus' Gradle task inputs and do not trigger an unnecessary rebuild.

Configuration properties specified via quarkus.quarkusBuildProperties or via the Quarkus application.* configuration files are all considered as Gradle task inputs, in other words: every change in these files causes a rebuild.

If your Quarkus build references system properties that do not start with quarkus. (or environment variables that do not start with QUARKUS_), you must reference those via the Quarkus build extension. For example, if your application.properties file references an environment variable like this:

greeting.message=${FOO_MESSAGE:Hello!}

it must be explicitly declared as "caching relevant":

quarkus {
   cachingRelevantProperties.add("FOO_MESSAGE")
   // Note: `cachingRelevantProperties` accepts regular expressions
}

fast-jar, jar

Dependency jars are stored unmodified as individual files in the quarkus-app/lib/ directory. All other files in the quarkus-app/ directory are generated.

The quarkusAppPartsBuild task builds fast-jar package-type applications and allows caching of the generated pieces, which is everything except the dependencies in the quarkus-app/lib/ directory. The quarkusDependenciesBuild task is used to collect the dependencies via Gradle mechanisms. The quarkusBuild task then assembles the outputs of the quarkusAppPartsBuild and quarkusDependenciesBuild tasks.

Note: fast-jar (or -jar) is the default if no package type has been explicitly configured.
